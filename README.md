# 🎴 بازی حکم (Hokm)

یک بازی چهار نفره‌ی حکم، به زبان فارسی، به‌صورت **PWA** (Progressive Web App) و تک‌فایلی — فقط با HTML/CSS/JS خالص، بدون هیچ فریم‌ورک یا بیلد تولزی.

شما (بازیکن ۱) در برابر سه هوش مصنوعی بازی می‌کنید؛ بازیکن ۳ یار شماست، بازیکن‌های ۲ و ۴ حریف.

---

## ✨ امکانات

- قوانین کامل حکم: حکم عادی، **نرس**، **آس‌نرس**، **سرس**
- تشخیص **کت** (۷-۰) و درخواست **بام** (ادامه تا ۱۳ دست)
- هوش مصنوعی مبتنی بر **شمارش کارت** (نه انتخاب تصادفی) — هر بازیکن AI بر اساس کارت‌های بازی‌شده و voidهای شناخته‌شده تصمیم می‌گیره:
  - قانون خاص «نفر سوم» (یار شروع‌کننده‌ی دست)
  - «کمترین کارت کافی» برای بقیه‌ی بازیکن‌ها
  - بریدن هوشمند بر اساس قطعیت بردن یار
  - لید زدن با حکم وقتی حریف‌ها طبق شمارش، بدون حکم شناخته شدن
  - ریسک‌پذیری متغیر بر اساس وضعیت کلی راند (عقب/جلو بودن)
- طراحی **موبایل‌فرست** با تم لیکویید گلس (Liquid Glass)، تطبیق خودکار با حالت روشن/تیره‌ی سیستم
- مناسب برای آیفون (پشتیبانی از safe-area / notch) و PC
- **ذخیره‌ی خودکار** بازی نیمه‌کاره در `localStorage` + امکان ادامه از همون‌جا
- **آمار کلی** بین بازی‌ها (تعداد برد، کت، بام)
- **تاریخچه‌ی راندها** در پایان هر بازی
- **مرور آخرین دست بازی‌شده** با یک دکمه
- افکت‌های صوتی سبک (Web Audio API، بدون فایل خارجی) + دکمه‌ی قطع/وصل صدا
- دکمه‌ی ریست کامل بازی

## 📂 ساختار پروژه

پروژه فقط یک فایله:

```
index.html   → کل بازی: HTML + CSS + JS در یک فایل
```

علاوه بر این، برای PWA بودن به این فایل‌ها نیاز داره (در این ریپو موجود نیستن، باید خودتون بسازید):

```
manifest.json     → مانیفست PWA
sw.js             → Service Worker (برای کارکرد آفلاین)
icon-192.png      → آیکون اپ
favicon.ico       → آیکون تب مرورگر
```

> ⚠️ مسیرها در `index.html` به‌صورت مطلق و هاردکد روی `/hokm/...` نوشته شدن (مثل `/hokm/manifest.json`، `/hokm/sw.js`). اگه پروژه رو جای دیگه‌ای دیپلوی می‌کنید (نه دقیقاً روی `دامنه.com/hokm/`)، باید این مسیرها رو اصلاح کنید.

## 🚀 اجرا

چون فایل تک‌صفحه‌ایه، هیچ نصب یا بیلدی لازم نیست:

1. فایل `index.html` رو مستقیم تو مرورگر باز کنید، **یا**
2. برای تست کامل PWA (نصب روی گوشی، آیکون، افلاین)، فایل رو روی یه وب‌سرور ساده سرو کنید:

```bash
# مثلا با پایتون
python3 -m http.server 8000
```

سپس از `http://localhost:8000` باز کنید.

برای نصب روی آیفون: از سافاری وارد سایت بشید → دکمه‌ی Share → «Add to Home Screen».

## 🧠 منطق هوش مصنوعی (خلاصه)

فایل `index.html` تابع `aiPick(pid)` رو داره که تصمیم هر بازیکن AI رو مشخص می‌کنه. نکات کلیدی:

| موقعیت بازیکن | رفتار وقتی می‌تونه ببره |
|---|---|
| نفر دوم / نفر چهارم | **کمترین کارت کافی** برای بردن |
| نفر سوم (یار شروع‌کننده‌ی دست) | **بزرگ‌ترین کارت** همون خال (یا می‌بره، یا نفر چهارم رو مجبور به خرج کارت باارزش می‌کنه) |
| وقتی یار در حال بردنه | اگه طبق شمارش کارت قطعیه → کارت بی‌ارزش می‌ده؛ اگه نامشخصه → بر اساس وضعیت راند ریسک می‌کنه |
| شروع یک دست جدید | اگه هر دو حریف طبق شمارش بدون حکم باشن، با خیال راحت پشت‌سرهم حکم می‌زنه |

تابع‌های کمکی مرتبط: `unaccountedHigherRanks`, `suitThreatExists`, `trumpCutThreatExists`, `isSecureWin`, `riskAppetite`.

## 🔧 نکات فنی برای توسعه‌ی بیشتر

- **تست سینتکس سریع:** چون همه‌چی تو یه `<script>` تگ داخل HTML‌ـه، برای چک کردن سریع خطای جاوااسکریپت می‌تونید محتوای اسکریپت رو استخراج و با `node --check` تست کنید.
- **ذخیره‌سازی:** کلیدهای `localStorage` استفاده‌شده:
  - `hokm_save_v1` → وضعیت بازی نیمه‌کاره
  - `hokm_stats_v1` → آمار کلی (برد/باخت/کت/بام)
  - `hokm_sound_v1` → وضعیت روشن/خاموش بودن صدا
- **افکت صوتی:** با `Web Audio API` و اسیلاتور ساخته می‌شه، بدون فایل صوتی خارجی.

## 💡 ایده‌های آینده (اضافه‌نشده)

- بازپخش کامل یک راند (نه فقط آخرین دست)
- پنل دیباگ برای دیدن دلیل تصمیم هر AI (برد قطعی / ریسک / بریدن)
- سطح سختی قابل‌انتخاب برای حریف
- خروجی گرفتن از نتیجه‌ی بازی به‌صورت عکس برای اشتراک‌گذاری
